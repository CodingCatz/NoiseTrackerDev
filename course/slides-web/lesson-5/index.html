這一課的關鍵做法

關卡，寫成「資料」而不是程式

  • 不寫死座標　哪裡有平台、缺口、起點終點，寫成一份「關卡資料」
  • 系統照著蓋　一個專門的程式讀這份資料，把整個世界生出來
  • 改關卡只改資料　之後想搬平台、加缺口，動資料就好，不用碰程式

這是第 1 課「資料驅動」心法的又一次實戰。

PHASE 09

蓋一個關卡

一個可以實際玩、而且用得上每一個招式的關卡。

▍你對 AI 說

💬 你對 AI 說

幫我蓋一個可以實際玩的關卡，而且要資料驅動——把哪裡有平台、缺口、起點終點寫成一份「關卡資料」，再由程式照著它把場景蓋出來。

不要把平台座標一個一個寫死在遊玩畫面裡，之後我才好改關卡 關卡至少要有：起點、高低不同的平台、一段要二段跳的高台、一段要衝刺的缺口、一段要牆跳的垂直通道、還有終點

你叫 AI 寫的「關卡資料」長這樣

一張圖，就是一份清單

座標用「單位」，(x, y) 是左上角、寬高用 w、h；注意 y 越大越靠下。

出生點 = { x:1, y:6 }
平台 = [
  { x:0,  y:7, w:8, h:2 }   地面
  { x:11, y:3, w:3, h:1 }   高台（要二段跳）
]
終點 = { x:13, y:5, w:1, h:2 }

關卡是刻意設計的

每段地形，都逼你用上一個招式

  • 起點 + 高低平台　暖身，熟悉走和跳
  • 高台　不用二段跳，上不去
  • 水平缺口　不用衝刺，跨不過
  • 垂直通道　不用牆跳，爬不上
  • 終點　抵達即通關

PHASE 10

鏡頭跟拍

讓鏡頭跟著角色跑，但別跑出世界外。

▍你對 AI 說

💬 你對 AI 說

幫我讓鏡頭跟著角色跑。

跟拍要平滑一點，不要硬邦邦地黏著角色、晃來晃去 鏡頭不可以拍到關卡外面——走到邊緣時，鏡頭就停在世界邊界 角色死掉或重重落地時，給鏡頭一點輕微震動，增加打擊感

鏡頭的兩條規矩

規矩 ①

平滑跟拍

鏡頭柔和地追上角色，而不是僵硬黏死、跟著一抖一抖，看久了會暈。

規矩 ②

不出邊界

走到關卡邊緣，鏡頭停在世界邊界，不露出關卡外面的空白。

第一次，
它像一個遊戲了。

本課小結

有世界、有鏡頭、
有可以挑戰的地形。

關卡資料驅動、每段地形對應一個招式、鏡頭平滑跟拍又守住邊界。下一課，我們加入讓世界「活起來」的互動：鑰匙、門、開關、會殺死你的尖刺、會復活你的存檔點。
