AI 協作開發實戰 · 第 0 課

課程導論

從一個點子，到一個能分享的網址——這門課要帶你走完的，是一整套用 AI 把想法變成上線產品的方法。

我們要從完全空白，
一路做到公開上線。

你會做出什麼

一款真的能玩的網頁遊戲

  • 左右奔跑、可變跳高、二段跳
  • 空中衝刺、貼牆下滑、牆跳
  • 踩到尖刺立刻重生、抵達終點即通關
  • 跑在你自己的網址上，丟連結就能玩

重點不是教你寫每一行程式，
是教你跟 AI 協作。

課程結束時，你會擁有

三樣帶得走的東西

  • 一個能玩的遊戲　放進作品集、丟給朋友
  • 一段公開的開發紀錄　GitHub 上十幾次提交，看得到專案怎麼長出來
  • 一套可複製的協作方法論　換任何專案、任何語言都適用

四個心法，
決定你用 AI 是事半功倍
還是越幫越忙。

心法 ①②

心法 ①

任務要小

不要說「幫我做一款遊戲」。要說「只完成這一件事，做完就停」。任務越小，AI 越不失控，你越好檢查。

心法 ②

每步都驗收

AI 給你程式碼，不代表能跑。每做完一小步就建置一次、親眼看一次。沒驗證過的「完成」等於沒完成。

心法 ③④

心法 ③

規格先行

不邊想邊做。先寫一份規格把「做什麼、數值多少、檔案怎麼擺」講清楚，再讓 AI 一段一段做。

心法 ④

留下紀錄

每完成一階段就用 Git 提交一次。AI 協作最怕的不是出錯，是錯了卻退不回去。

準備工作

你只需要四樣工具

  • 一個 AI 編碼助手　能讀檔、改檔、幫你跑指令的那種
  • Node.js　跑開發環境
  • Git + GitHub 帳號　免費，存程式碼與上架
  • 一個瀏覽器　拿來玩你自己做的遊戲
node -vgit --version看到版本號＝準備好了
