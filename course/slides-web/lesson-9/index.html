請用 GitHub 官方的 Pages 功能來做 注意：Pages 網址會多一層專案名稱的路徑，請把專案設定調對，免得上線後變白畫面、圖載不出來

這套機制叫 GitHub Actions。設定好一次，以後每次更新都自動上線——你只管寫程式。

新手一定會踩的坑：白畫面

✕ 沒設對

上線後一片白

畫面全白、圖片全部載不出來。因為網址多了一層「專案名稱」，程式卻去舊路徑找資源。

✓ 設對了

告訴它正確路徑

讓專案知道「網址多了一層專案名稱」，所有圖片、程式就都找得到了。

規格裡那段「自動抓專案名稱」的設定，就是替你避開這個坑。

把開關打開

四步，正式上線

  • 把程式碼推上 GitHub　git add . → git commit → git push
  • 倉庫 Settings → Pages → Source　選「GitHub Actions」
  • 回 Actions 分頁，等流程跑完　看到綠色的勾
  • 點開網址　你的遊戲，全世界都能玩

看到綠勾，點開網址——
你的遊戲上線了。

更新 README：這是什麼遊戲、怎麼操作、有哪些能力、怎麼開發、怎麼部署、還有遊戲網址 建立三份紀錄表：我下過哪些指令、試玩測試紀錄、調手感的過程

請照專案真實的樣子寫，不要寫成廣告、也別假裝做了還沒做的功能。

你的課程交付物

四份文件，交作業也用得上

  • README　專案說明書：是什麼、怎麼玩、怎麼跑、網址在哪
  • 指令紀錄　你這趟下過哪些指令——一本 AI 協作的「航海日誌」
  • 試玩紀錄　玩過哪些、發現什麼問題
  • 調校筆記　跳躍手感怎麼一步步調出來的

你真正帶走的

不是這款遊戲，是這套方法

  • 先寫規格　給 AI 一張看得懂的施工藍圖
  • 把大象切成片　一次只做一件事
  • 每一步都驗收　沒驗過的「完成」等於沒完成
  • 隨時能退回　每步提交，錯了退得回去

這套方法跟平台遊戲無關——它是你未來跟 AI 做任何東西的底層作業系統。
